Examples

DataTalksClub contact lifecycle

curl -sS -X POST "$DATAMAILER_URL/api/v1/contacts" \
  -H "Authorization: Bearer $DATAMAILER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "learner@example.com",
    "audience": "datatalksclub",
    "client": "dtc-courses",
    "status": "subscribed",
    "tags": ["ml-zoomcamp"],
    "verified": false
  }'

curl -sS "$DATAMAILER_URL/api/v1/contacts/status?email=learner@example.com&audience=datatalksclub&client=dtc-courses" \
  -H "Authorization: Bearer $DATAMAILER_API_KEY"

curl -sS -X PATCH "$DATAMAILER_URL/api/v1/contacts/{contact_id}/verification" \
  -H "Authorization: Bearer $DATAMAILER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"audience":"datatalksclub","client":"dtc-courses","verified":true}'

curl -sS "$DATAMAILER_URL/api/v1/contacts?audience=datatalksclub&client=dtc-courses&tags=ml-zoomcamp&limit=100" \
  -H "Authorization: Bearer $DATAMAILER_API_KEY"

AI Shipping Labs transactional and import/export

import requests

base_url = "http://localhost:8000"
headers = {"Authorization": f"Bearer {datamailer_api_key}"}

requests.post(
    f"{base_url}/api/v1/transactional/send",
    headers=headers,
    json={
        "email": "user@example.com",
        "template_key": "account-verification",
        "idempotency_key": "client-generated-id",
        "context": {"verification_url": "https://example.com/verify"},
        "metadata": {"source": "account-flow"},
    },
    timeout=10,
)

requests.post(
    f"{base_url}/api/v1/contacts/imports",
    headers=headers,
    json={
        "audience": "ai-shipping-labs",
        "client": "ai-shipping-labs",
        "dry_run": False,
        "contacts": [
            {
                "email": "subscriber@example.com",
                "status": "subscribed",
                "tags": ["newsletter"],
                "email_validation": {"status": "externally_validated"},
            }
        ],
    },
    timeout=10,
)

requests.get(
    f"{base_url}/api/v1/contacts.csv",
    headers=headers,
    params={"audience": "ai-shipping-labs", "client": "ai-shipping-labs"},
    timeout=10,
)

Notes

Transactional sending uses provisioned transactional templates. API-based template catalog management is planned separately.

Public tracking, unsubscribe, and SES webhook routes are documented for integration context; client application endpoints under /api/v1 use Bearer authentication.
